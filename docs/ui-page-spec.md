# 个人 AI 数字分身网站 UI 风格规范与页面规格

## 1. 产品定位

本项目是一个面向访客的个人 AI 数字分身网站，同时包含管理员后台。

访客可以浏览个人介绍、学习成长、生活记录、工作/项目经历、照片内容，并通过聊天框与“虚拟的我”对话。管理员可以维护个人资料、知识库、照片与生活记录、访客问题、聊天记录、大模型配置和提示词策略。

网站需要强关联站点主人的学习、生活、工作三条主线，帮助其在升学、社交、求职场景中更好地展示自己、解释自己、连接他人。

核心关键词：

- 成熟但有活力
- 年轻但不幼稚
- 个人叙事感
- 学习、生活、工作的成长档案
- AI 陪伴感
- 照片优先的信息流体验
- 可持续沉淀的个人知识库

## 1.1 核心使用场景

### 1.1.1 升学场景

目标访客可能是导师、招生老师、项目负责人、同学或推荐人。

网站需要呈现：

- 学习经历
- 课程与成绩亮点
- 研究兴趣
- 竞赛、证书、论文、项目
- 学习方法和长期成长轨迹
- 能体现自驱力、好奇心和持续投入的生活记录

### 1.1.2 求职场景

目标访客可能是 HR、面试官、团队负责人、合作方。

网站需要呈现：

- 工作经历
- 项目经验
- 技能栈
- 作品集
- 实习经历
- 职业目标
- 解决问题的方式
- 可下载或可查看的简历信息

### 1.1.3 社交场景

目标访客可能是朋友、同学、兴趣圈层、潜在合作伙伴。

网站需要呈现：

- 生活记录
- 兴趣爱好
- 旅行、阅读、运动、创作等片段
- 价值观和偏好
- 可自然开启话题的照片与记录
- 让别人更容易了解你、接近你、与你产生连接的内容

## 2. 设计原则

### 2.1 前台访客端

前台页面需要让访客感到轻松、鲜活、愿意继续浏览。

设计原则：

- 首页直接呈现“我是谁”和“可以和虚拟的我对话”。
- 页面内容围绕学习、生活、工作三条主线展开。
- 生活记录采用朋友圈 / 小红书式浏览体验，但不复制具体平台品牌样式。
- 照片优先，文字辅助，详情页承载完整故事。
- AI 聊天入口应在首页、生活记录、详情页中反复出现。
- 页面需要有年轻感，但主结构保持成熟、清晰、可信赖。

### 2.2 管理员后台

后台页面需要高效、克制、稳定。

设计原则：

- 信息密度高于前台，但不拥挤。
- 左侧固定导航，顶部显示当前页面状态和关键操作。
- 表格、筛选、编辑器、上传组件、开关组件统一风格。
- 青绿色作为主要操作色，其他活力色只用于状态、标签和提醒。
- 后台不做社交化装饰，强调维护效率和数据沉淀。

## 3. 色彩规范

### 3.1 主色

| 用途 | 色值 | 说明 |
|---|---:|---|
| 主色 Mature Teal | `#00A88A` | 主按钮、选中态、主要链接、关键操作 |
| 主色深态 | `#007D6C` | Hover、Pressed、强调标题 |
| 主色浅底 | `#E5F8F4` | 选中菜单、标签背景、轻提示 |

### 3.2 活力辅助色

| 用途 | 色值 | 说明 |
|---|---:|---|
| 珊瑚强调 | `#FF6B5F` | 高价值提醒、待处理、重要入口 |
| 青柠点缀 | `#B7E75A` | 正向状态、轻快标签、增长提示 |
| 天蓝辅助 | `#4CB9E7` | 次级信息、AI 关联、推荐问题 |
| 阳光黄 | `#FFC83D` | 少量用于灵感、生活分类、温暖提醒 |

### 3.3 中性色

| 用途 | 色值 |
|---|---:|
| 页面背景 | `#F7F9F8` |
| 卡片背景 | `#FFFFFF` |
| 主文字 | `#1F2528` |
| 次级文字 | `#6F7D7A` |
| 弱文字 | `#A0AAA7` |
| 边框 | `#DDE8E4` |
| 分割线 | `#EDF2F0` |

### 3.4 使用规则

- 大面积背景使用 `#F7F9F8` 或白色。
- 主操作统一使用成熟青绿。
- 珊瑚色只用于需要被注意的内容，避免页面变得焦躁。
- 青柠、天蓝、阳光黄只作为标签、状态、局部高光。
- 禁止大面积紫色、深蓝、灰蓝或单一米色主题。
- 前台可以更活泼，后台必须更克制。

## 4. 字体与排版

推荐字体：

- 中文：`Inter`, `PingFang SC`, `Microsoft YaHei`, `sans-serif`
- 数字与英文：`Inter`, `SF Pro Display`, `Arial`

字号建议：

| 场景 | 字号 | 字重 |
|---|---:|---:|
| 页面主标题 | 32-40px | 700 |
| 页面二级标题 | 24-28px | 700 |
| 卡片标题 | 16-20px | 600 |
| 正文 | 14-16px | 400 |
| 辅助信息 | 12-13px | 400 |
| 按钮文字 | 14-15px | 600 |

排版规则：

- 不使用负字距。
- 不用随视口宽度变化的字体尺寸。
- 页面标题只在真正的首页或大页面区域使用大字号。
- 后台表格、卡片、侧栏使用紧凑但可读的字号。

## 5. 圆角、阴影与间距

圆角：

- 普通卡片：`8px`
- 输入框 / 按钮：`8px`
- 图片卡片：`10px`
- 标签：`999px`

阴影：

- 前台卡片可使用轻阴影：`0 8px 24px rgba(31, 37, 40, 0.06)`
- 后台卡片尽量使用边框，少用阴影。

间距：

- 页面左右边距：桌面端 40-64px，移动端 16px。
- 卡片内边距：16-24px。
- 后台表格行高：48-56px。
- 生活记录瀑布流间距：16-20px。

## 7. 前台页面规格

### 7.1 首页 `/`

目标：

- 让访客快速知道你是谁。
- 明确引导访客与虚拟的你对话。
- 展示学习、生活、工作三条主线和个人亮点。

模块：

- 顶部导航：生活记录、关于我、AI 对话、联系我。
- 个人简介区：头像、昵称、一句话介绍、关键词标签。
- 主行动按钮：`开始咨询`、`查看生活记录`。
- 三条主线入口：
  - 学习成长
  - 生活记录
  - 工作项目
- 最近更新：展示 4-6 条生活记录卡片。
- 代表项目：展示 2-3 个最能体现能力的项目。
- 学习亮点：课程、竞赛、证书、研究兴趣或阶段性成果。
- 推荐问题：引导访客提问。
- 个人亮点：能力、经历、兴趣、当前关注。

主要流转：

- 点击 `开始咨询` -> `/chat`
- 点击生活记录卡片 -> `/life/:id`
- 点击学习成长 -> `/study`
- 点击工作项目 -> `/work`
- 点击 `关于我` -> `/about`

### 7.2 AI 咨询页 `/chat`

目标：

- 让访客通过聊天框与“虚拟的我”对话。
- AI 回答优先基于个人知识库。
- 保存访客问题和回答，用于后续沉淀。

模块：

- 聊天标题区：说明这是 AI 分身，不是真人实时在线。
- 推荐问题区：常见问题、关于经历、关于生活记录、联系方式。
- 消息区：访客消息、AI 回复、引用来源。
- 输入区：多行输入、发送按钮。
- 关联内容区：推荐查看的生活记录或关于我内容。

状态：

- 初始空状态
- 正在生成
- 回答成功
- 知识库无结果
- API 异常
- 访客输入过长

主要流转：

- 访客提问 -> 检索知识库 -> 调用大模型 -> 返回回答 -> 保存聊天记录
- 点击引用内容 -> `/life/:id` 或 `/about`

### 7.3 生活记录信息流 `/life`

目标：

- 用类似朋友圈 / 小红书的舒适浏览体验呈现照片和生活记录。
- 让生活记录自然成为个人知识库的一部分。

模块：

- 顶部标题：`生活记录`
- 分类筛选：全部、旅行、日常、成长、灵感、学习、工作。
- 搜索入口。
- 瀑布流卡片：
  - 图片
  - 标题
  - 短文案
  - 时间
  - 地点
  - 标签
  - `查看详情`
  - `问问虚拟的我`

主要流转：

- 点击卡片 -> `/life/:id`
- 点击 `问问虚拟的我` -> `/chat?recordId=xxx`
- 点击标签 -> 筛选当前标签内容

### 7.4 生活记录详情 `/life/:id`

目标：

- 展示单条生活记录的完整故事。
- 让访客可以围绕这条记录继续问 AI。

模块：

- 图片轮播或大图。
- 标题。
- 时间、地点、标签。
- 故事正文。
- 相关记忆。
- 相关照片。
- `问问虚拟的我` 按钮。

主要流转：

- 点击 `问问虚拟的我` -> `/chat?recordId=xxx`
- 点击相关记忆 -> `/life/:id`

### 7.5 关于我 `/about`

目标：

- 展示更完整的个人资料、经历、能力、价值观。

模块：

- 基本信息。
- 一句话介绍。
- 经历时间线。
- 技能能力。
- 兴趣偏好。
- 当前关注。
- 联系入口。

主要流转：

- 点击 `与虚拟的我对话` -> `/chat`
- 点击 `联系我` -> `/contact`

### 7.6 学习成长 `/study`

目标：

- 服务升学、导师了解、同学协作等场景。
- 展示学习轨迹、研究兴趣、课程能力、竞赛证书和成长过程。

模块：

- 学习概览：学校、专业、阶段、方向。
- 研究兴趣 / 学习方向。
- 课程与能力标签。
- 竞赛、证书、论文、活动。
- 学习记录时间线。
- 代表学习成果。
- 与学习相关的生活记录。
- `问问虚拟的我`：围绕学习经历提问。

主要流转：

- 点击学习成果 -> `/study/:id` 或 `/work/:id`
- 点击相关记录 -> `/life/:id`
- 点击 `问问虚拟的我` -> `/chat?topic=study`

### 7.7 工作项目 `/work`

目标：

- 服务求职、实习、合作、项目展示等场景。
- 展示技能、经历、项目成果和职业方向。

模块：

- 职业目标。
- 技能栈。
- 项目作品列表。
- 实习 / 工作经历。
- 可下载简历入口。
- 项目详情入口。
- 相关问题推荐。

主要流转：

- 点击项目 -> `/work/:id`
- 点击简历 -> 下载或预览简历
- 点击 `问问虚拟的我` -> `/chat?topic=work`

### 7.8 项目详情 `/work/:id`

目标：

- 用结构化方式说明一个项目，方便升学和求职场景引用。

模块：

- 项目标题。
- 项目时间。
- 个人角色。
- 背景问题。
- 解决方案。
- 技术栈 / 方法。
- 成果与数据。
- 项目截图 / 图片。
- 相关知识库引用。
- `问问虚拟的我`。

### 7.9 联系我 `/contact`

目标：

- 给访客提供明确、可信的联系路径。

模块：

- 邮箱 / 微信 / 社交账号。
- 留言表单。
- 简历下载或求职联系入口。
- 升学 / 合作 / 社交三类联系意图选择。
- 常见联系问题。
- AI 咨询入口。

## 9. 页面流转关系

### 9.1 访客浏览流转

```text
首页 /
  -> AI 咨询 /chat
  -> 生活记录 /life
  -> 生活记录详情 /life/:id
  -> 学习成长 /study
  -> 工作项目 /work
  -> 项目详情 /work/:id
  -> 关于我 /about
  -> 联系我 /contact
```

### 9.2 AI 咨询流转

```text
访客提问
  -> 聊天接口
  -> 检索个人知识库
  -> 组装 Prompt
  -> 调用 DeepSeek API
  -> 返回回答
  -> 保存聊天记录
  -> 进入访客问题池
```

### 9.3 生活记录沉淀流转

```text
上传图片
  -> 创建生活记录
  -> 设置公开展示
  -> 设置加入知识库
  -> 生成知识条目
  -> 可被 AI 检索引用
```

### 9.4 访客问题沉淀流转

```text
访客问题
  -> 后台问题池
  -> 管理员判断价值
  -> 整理标准回答
  -> 加入知识库
  -> 标记已沉淀
```

## 11. 开发注意事项

- 前台和后台使用同一套设计 Token，但后台降低活力色使用比例。
- 生活记录中的图片、正文、标签可以同步生成知识库条目。
- AI 回答必须优先基于知识库，不允许编造个人经历。
- 聊天记录和访客问题必须保存，方便后续沉淀。
- 所有公开内容都需要管理员可控。
- 图片上传需要压缩、预览、删除和公开状态控制。
- 后台所有危险操作需要二次确认。
- API Key 必须加密或至少不在前端暴露。
