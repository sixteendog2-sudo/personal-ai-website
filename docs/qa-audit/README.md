## 修复回归

- 访客端导航已移除后台入口；后台页面设置为禁止搜索引擎索引。
- 未登录访问后台 API 仍返回 `401`，后台页面继续由管理员会话保护。
- 后台窄视口默认显示主内容，导航改为可通过键盘操作的抽屉菜单。
- 登录页不再展示任何默认密码或具体凭据。
- AI 回复中的标题、列表、粗体和行内代码已安全渲染。
- 生活分类已改为可操作按钮，提供选中状态和空结果反馈。
- 工作、生活、学习后台已接通新增、编辑、永久删除；生活和工作支持图片上传。
- 内容保存会同步纯文本知识，图片与媒体元数据不会进入模型上下文。
- 知识库支持新增、编辑、重新分块和停用，访客问题转换后可继续维护。

## 可访问性风险

- 需要补充键盘完整走查：导航、推荐问题、聊天发送、后台菜单、上传和保存流程的焦点顺序及可见焦点。
- 浅绿色标签和灰色辅助文本需要用对比度工具实测；仅凭截图不能确认是否达到 WCAG 对比度要求。
- 后台响应式问题也会影响放大页面、窄窗口和部分辅助技术用户。
- 生活分类如果承担筛选功能，应使用原生按钮，并提供 `aria-pressed` 或等价的当前状态信息。

## 证据边界

本报告基于本次本地运行的页面截图、DOM 结构、接口响应、数据库记录和自动化测试。截图审计不能证明完整 WCAG 合规，也未覆盖真实移动设备、不同浏览器、弱网、长时间并发和第三方部署环境。文件上传已通过接口验证，未在内置浏览器中重复提交真实文件。
