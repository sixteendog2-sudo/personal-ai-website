# 个人 AI 数字分身网站使用手册

适用版本：当前本地项目
更新日期：2026-06-21

## 1. 项目定位

这是一个面向升学、社交、求职和项目合作场景的个人 AI 网站。它把个人内容分成三条主线：

- 学习成长：研究兴趣、课程、证书、学习笔记和阶段成果。
- 生活记录：日常、旅行、成长与灵感记录。
- 工作项目：项目经历、个人角色、技术栈和成果。

访客既可以浏览这些公开内容，也可以向“虚拟的我”提问。AI 只检索允许公开和引用的知识，并把访客问题、回答、引用来源和模型调用结果沉淀到后台。

## 2. 用户角色与入口

| 角色 | 入口 | 能做什么 |
| --- | --- | --- |
| 访客 | `http://127.0.0.1:3000` | 浏览公开内容、使用 AI、提交联系意向 |
| 管理员 | `http://127.0.0.1:3000/admin` | 查看运营数据、导入学习资料、维护知识闭环、查看日志 |
| 健康检查 | `http://127.0.0.1:3000/api/health` | 查看服务和数据库是否正常 |

访客端不展示后台入口。直接访问后台时，未登录用户会被送到管理员登录页；未登录调用后台 API 会返回 `401`。

## 3. 功能状态说明

本手册使用以下三种状态：

- **可用**：页面和数据操作已经接通，可以正常使用。
- **只读**：页面会读取真实数据，但当前没有完整编辑界面。
- **展示型**：已有界面样式，但按钮尚未连接保存逻辑。
- **API 已具备**：后端接口已完成，但后台页面尚未接入该接口。

## 5. 首次安装

### 5.1 环境要求

- Windows PowerShell
- Node.js `>=20.18`
- pnpm `11`
- PostgreSQL `17`
- Git
- 安装 pgvector 时还需要 Visual Studio C++ Build Tools

### 5.2 安装依赖

在项目目录打开 PowerShell：

```powershell
Set-Location -LiteralPath "E:\个人网页"
pnpm install
```

### 5.3 创建本地配置

复制 `.env.example` 为 `.env.local`，至少填写：

```env
DATABASE_URL=postgresql://personal_ai_app:数据库密码@127.0.0.1:5432/personal_ai
DATABASE_MAX_CONNECTIONS=10
DATABASE_SSL=false

ADMIN_EMAIL=你的管理员邮箱
ADMIN_PASSWORD=至少12位的强密码
ADMIN_SESSION_TOKEN=至少32位的随机字符串

STORAGE_DRIVER=local
```

`.env.local` 已被 Git 忽略。不要把数据库密码、管理员密码、DeepSeek Key 或 S3 密钥提交到仓库。

### 5.4 安装并启用 pgvector

如果本机尚未安装 pgvector，请在“管理员 PowerShell”中运行：

```powershell
Set-Location -LiteralPath "E:\个人网页"
.\scripts\install-pgvector-windows.ps1
.\scripts\bootstrap-pgvector-database-windows.ps1
```

第一个脚本编译并安装 pgvector；第二个脚本为目标数据库启用 `vector` 扩展，并安全恢复临时修改的 PostgreSQL 认证配置。

### 5.5 初始化数据库和管理员账号

```powershell
pnpm db:migrate
pnpm db:seed
```

`db:seed` 会使用 `.env.local` 中的 `ADMIN_EMAIL` 和 `ADMIN_PASSWORD` 创建或更新管理员账号，同时写入示例内容、默认知识、模型配置和提示词。

如果修改了管理员密码，应再次执行 `pnpm db:seed`，让数据库中的密码哈希同步更新。

### 5.6 配置 DeepSeek

推荐使用隐藏输入脚本：

```powershell
pnpm ai:configure:windows
```

输入 Key 后，脚本会更新：

- `DEEPSEEK_API_KEY`
- `DEEPSEEK_BASE_URL=https://api.deepseek.com`
- `DEEPSEEK_CHAT_MODEL=deepseek-chat`

配置后必须重启开发服务。

### 5.7 启动项目

```powershell
pnpm dev
```

然后访问：

- 访客端：`http://127.0.0.1:3000`
- 后台：`http://127.0.0.1:3000/admin`
- 健康检查：`http://127.0.0.1:3000/api/health`

## 6. 日常使用流程

### 6.1 发布一条学习记录

1. 进入 `/admin` 并登录。
2. 打开“学习档案”。
3. 选择 `.md`、`.markdown` 或 `.txt` 文件，或者直接填写正文。
4. 填写标题和访问路径。访问路径只能使用小写字母、数字和连字符，例如 `ai-product-notes`。
5. 选择类型、学习周期、来源和标签。
6. 设置状态：
   - 草稿：不作为正式公开内容。
   - 发布：允许进入公开内容流程。
7. 设置可见范围：
   - 私密：仅后台使用。
   - 不公开列出：不出现在公开列表中。
   - 公开：访客可以浏览。
8. 只有确实允许公开引用时，才勾选“允许 AI 在回答中引用”。
9. 点击“保存学习记录”。

要让一条学习记录同时出现在访客列表并被 AI 引用，建议同时满足：`发布 + 公开 + 允许 AI 引用`。

### 6.2 处理访客问题

1. 访客在 `/chat` 提问。
2. 系统保存会话、问题、AI 回答和引用来源。
3. 管理员进入“访客问题”。
4. 对值得长期保留的问题点击“问题转知识”。
5. 系统在同一个数据库事务中创建知识条目，并把问题状态改为 `converted`。
6. 后续访客提问时，新知识可以被检索和引用。

注意：转换操作会直接生成 `公开 + 已发布 + AI 可用` 的知识条目。转换前应确认回答中没有隐私或错误信息。

### 6.3 查看联系意向

1. 访客在 `/contact` 提交联系表单。
2. 管理员进入“联系意向”。
3. 查看联系目的、联系方式和留言。

访客联系方式属于敏感数据，不应复制到公开知识库或公开页面。

### 6.4 验证 AI 是否工作

1. 打开 `/chat`。
2. 提问一个能在公开知识中找到答案的问题。
3. 检查回答下方是否出现引用来源。
4. 进入后台“聊天记录”和“访客问题”，确认记录已写入。
5. 如果回答提示真实模型失败，检查 DeepSeek 配置和网络；系统会自动降级为本地知识库回答。

### 6.5 备份数据库

```powershell
pnpm db:backup:windows
```

备份文件写入 `backups/`，格式为 `personal-ai-日期时间.dump`。脚本会立即运行 `pg_restore --list` 验证备份可读。生产环境还应把备份复制到独立存储，并定期演练恢复。

## 7. 内容发布与 AI 引用规则

每条内容或知识至少涉及三个开关：

| 字段 | 典型值 | 作用 |
| --- | --- | --- |
| `status` | `draft` / `published` / `archived` | 是否处于正式发布状态 |
| `visibility` | `private` / `unlisted` / `public` | 谁可以看到 |
| `isAiUsable` | `true` / `false` | 是否允许 AI 检索和引用 |

访客 AI 检索会强制要求：

```text
status = published
visibility = public
isAiUsable = true
```

因此，单独勾选“AI 可用”并不会让私密或草稿内容泄露给访客。

## 8. AI 工作流程

```text
访客提问
  → 创建或复用聊天会话
  → 按主题检索公开知识
  → 生成知识上下文和引用
  → 调用 DeepSeek
  → 失败时降级为本地回答
  → 保存用户消息、AI 消息和调用日志
  → 保存访客问题
  → 管理员可将优质问题转为知识
```

模型提示词明确要求：知识库没有相关信息时应说明“暂时没有记录”，不得编造身份、学历、经历、项目、观点或生活故事。

### 9.2 后台 API

后台 API 位于 `/api/admin/*`，全部需要有效管理员会话。主要能力包括：

- 内容：列表、新建、详情、修改、删除、绑定媒体。
- 学习文档：上传或粘贴 Markdown/TXT 并创建学习内容。
- 知识库：列表、新建、详情、修改、删除。
- 访客问题：列表、转为知识。
- 媒体：列表、上传和生成缩略图。
- 个人资料：读取和修改。
- 模型设置：读取和激活新设置。
- 提示词：读取和发布新版本。
- 联系意向、仪表盘和审计日志：读取。

后台写接口会校验管理员账号仍处于启用状态，并拒绝跨站 Origin 请求。

## 10. 安全设计

- 访客端不显示后台入口，后台页面禁止搜索引擎索引。
- 管理员密码使用 bcrypt 哈希存储，不保存明文。
- 登录成功后使用签名 JWT，并存入 HttpOnly、SameSite=Lax Cookie。
- 会话有效期为 7 天；生产环境 Cookie 启用 Secure。
- 后台页面和后台 API 由统一代理保护。
- 后台写请求会再次核验数据库中的管理员启用状态。
- 联系和聊天接口有数据库持久化限流。
- 媒体上传验证真实文件类型，不只依赖扩展名。
- 本地存储会阻止目录穿越。
- 访客知识检索强制执行公开、发布和 AI 可用条件。

## 12. 常见问题

### 健康检查返回 503

检查：

1. PostgreSQL 服务是否启动。
2. `.env.local` 中的 `DATABASE_URL` 是否正确。
3. 数据库、用户和密码是否存在。
4. 是否已执行 `pnpm db:migrate`。

### 后台密码不正确

确认 `.env.local` 中的 `ADMIN_PASSWORD` 至少 12 位，然后执行：

```powershell
pnpm db:seed
```

重启服务后再次登录。

### DeepSeek 配置后仍使用本地回答

1. 重新运行 `pnpm ai:configure:windows`。
2. 重启 `pnpm dev`。
3. 检查网络是否能访问 `https://api.deepseek.com`。
4. 确认 Key 有效且额度正常。

### AI 没有引用新内容

确认该内容或知识同时为：

- `published`
- `public`
- `isAiUsable = true`

并确保正文中包含访客问题会使用的关键词。

### pgvector 脚本提示需要管理员权限

关闭当前窗口，右键 PowerShell 选择“以管理员身份运行”，再进入项目目录执行脚本。

### PowerShell 执行策略阻止脚本

可仅对本次进程放开：

```powershell
Set-ExecutionPolicy -Scope Process Bypass
```

### 为什么后台模型设置里不能填写 API Key

这是有意的安全设计。模型参数可以在后台保存，但 DeepSeek Key 只允许通过服务器环境变量 `DEEPSEEK_API_KEY` 配置，页面不会读取、回显或保存真实密钥。
